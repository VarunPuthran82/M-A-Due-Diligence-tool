# M&A Intelligence Audit Engine

The **M&A Due Diligence Engine** is an advanced, client-side web application designed for financial analysts, auditors, and executives. Its primary use case is to parse, analyze, and compare large corporate filings (such as 10-Ks, 10-Qs, and Annual Reports in PDF/HTML format) during Mergers & Acquisitions (M&A) due diligence.

Unlike traditional Retrieval-Augmented Generation (RAG) systems that slice documents into small chunks—which can lose broader context—this application leverages the massive context window of the **Gemini 3 Pro** model. It processes entire documents natively to deliver comprehensive, cross-document reasoning, identifying subtle language shifts, strategic contradictions, and overlapping risks between entities.

## 🚀 Features

*   **1M+ Token Context Window:** Upload multiple massive PDF or HTML filings directly into the browser and analyze them natively without chunking or vector databases.
*   **Deep Comparative Analysis:** Identify thematic clusters and contrast the "perceived severity" between multiple entities side-by-side. 
*   **Citation-Backed Precision:** Every figure, metric, or claim generated by the engine includes a strict page-level citation (e.g., `[Apple, Page 34]`), ensuring complete traceability.
*   **Executive-Ready Markdown:** Output is automatically parsed into clean, C-suite ready markdown tables, bullet points, and headers.
*   **Interruptible Streaming:** Live-streaming analysis that allows you to stop generation mid-flight if you want to retry or edit your prompt.
*   **Native Privacy:** Secure data room isolation. Documents are processed and sent directly to the Gemini API from the client.

## 🛠 Tech Stack

*   **Frontend:** React 19, Vite, TypeScript
*   **Styling:** Tailwind CSS (Glassmorphic UI)
*   **AI Engine:** Google Gemini SDK (`@google/genai`) using `gemini-3-pro-preview`

## ⚙️ How to Run Locally

Because the project directory name contains an ampersand (`&`), standard npm scripts might fail on Windows Command Prompt/PowerShell. Follow these steps:

1. **Clone the repository:**
   ```bash
   git clone https://github.com/VarunPuthran82/M-A-Due-Diligence-tool.git
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Configure the API Key:**
   Create a `.env.local` file in the root directory and add your Google Gemini API key:
   ```env
   GEMINI_API_KEY=your_actual_api_key_here
   ```

4. **Start the Development Server:**
   Bypass the Windows `&` script parsing bug by executing Vite directly:
   ```bash
   node node_modules\vite\bin\vite.js
   ```

5. **Access the App:** Open `http://localhost:3000` (or whichever port Vite assigns) in your browser.

## 📖 How to Use

1. **Import Data Room:** Click the import button to load your `.pdf`, `.html`, or `.txt` filings.
2. **Preset Audits:** Click one of the quick templates like "Capital Structure Snapshot" or "Item 1A Risk Audit".
3. **Custom Queries:** Use the chat terminal at the bottom to ask highly specific cross-document questions. *Example: "Compare the estimated Useful Life of Server Equipment between these two companies."*
4. **Refine:** Use the Edit/Retry buttons on your queries to iterate on your analysis, or hit Stop if you want to cancel a long-running generation.

## ⚠️ Notes for Developers
*   **Model:** This app uses `gemini-3-pro-preview`. Ensure your API key has access to these specific model tiers.
*   **Document Support:** Native text PDFs provide the fastest parsing and most accurate citations compared to scanned images.
